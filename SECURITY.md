# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues.

Instead, use a private reporting channel if one is available on the repository hosting platform. If private reporting is not available, contact the maintainers directly and include:

- a clear description of the issue
- impact and affected components
- reproduction steps or proof of concept
- any suggested mitigation if known

## What to expect

- We will review the report as quickly as possible
- We may ask follow-up questions to reproduce or validate the issue
- We will try to coordinate disclosure after a fix or mitigation is available

## Scope notes

This project interacts with live websites through a real browser. Reports involving unsafe browser state, exposed HTTP deployments, accidental credential leakage, or container/network misconfiguration are in scope if they materially affect the security of the project or its recommended deployments.
